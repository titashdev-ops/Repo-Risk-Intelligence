# Security Policy

## Supported Versions

Repo Risk Intelligence is currently in active MVP development.

Security fixes will be applied to the latest version of the project.

| Version | Supported |
|---------|-----------|
| MVP / main | ✅ |

---

## Reporting a Vulnerability

If you discover a security issue, please do **not** disclose it publicly before it has been reviewed.

Instead:

1. Prepare a clear description of the issue.
2. Include steps to reproduce when possible.
3. Describe the potential impact.

Until a dedicated security contact is established, please open a private discussion or contact the project owner through GitHub.

---

## Security Principles

- Protect user data.
- Follow the principle of least privilege.
- Validate inputs.
- Avoid exposing sensitive information.
- Address security issues before adding new features.

---

## Responsible Disclosure

Please allow reasonable time to investigate and resolve reported vulnerabilities before public disclosure.

Thank you for helping improve the security of Repo Risk Intelligence.
